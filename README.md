# SkyFitnessPro — Дипломный проект

## 📖 Описание
Веб-приложение для фитнес-платформы, позволяющее пользователям просматривать курсы, проходить тренировки, отслеживать прогресс и управлять своим профилем. Разработано в рамках дипломного проекта.


## 🛠 Стек технологий
- Frontend: Next.js (Pages Router), React, TypeScript
- Стилизация: styled-components
- База данных: MongoDB (драйвер `mongodb`) — локально
- Аутентификация:
    - JWT-токены, localStorage, React Context
    - Внешний API: `https://wedev-api.sky.pro/api/user`

- Работа с данными:
    - Курсы/тренировки: внешний API `https://wedev-api.sky.pro/api/fitness`
    - Пользователи/прогресс: локальная MongoDB или внешний API

- Особенность: Запросы к учебному API должны отправляться **без заголовка** `Content-Type: application/json`
- Деплой: Cloudflare Pages (настройка в процессе)
- Инструменты: Git, npm, axios (для авторизации), fetch (для курсов)

## 🔐 Архитектура авторизации
Проект использует **два разных базовых URL** для разных типов запросов:

| Тип запроса                  | Базовый URL                             | Файл-клиент         | Особенности                                      |
| ---------------------------- | --------------------------------------- | ------------------- | ------------------------------------------------ |
| 📚 Курсы, тренировки         | `https://wedev-api.sky.pro/api/fitness` | `libs/apiConfig.ts` | Использует `fetch`, заголовок `Content-Type: ""` |
| 🔐 Авторизация, пользователи | `https://wedev-api.sky.pro/api`         | `libs/apiAuth.ts`   | Использует `axios`, заголовок `Content-Type: ""` |

### ⚠️ Важное требование API
Учебный сервер **не принимает** заголовок `Content-Type: application/json`. Все запросы должны отправляться либо с пустым заголовком, либо без него вовсе:

````typescript
// Правильно:
headers: { "Content-Type": "" }

// Неправильно (вернёт 400 ошибку):
headers: { "Content-Type": "application/json" }
````

## 🎨 Страницы авторизации и регистрации
### Архитектура модалок

Формы входа и регистрации реализованы как **модальные окна поверх главной страницы**, а не как отдельные маршруты. Это обеспечивает:
- Плавный UX без перезагрузки страницы
- Сохранение состояния курсов и фильтров
- Единый визуальный контекст приложения

**Механизм работы:**
- Ссылки в шапке ведут на `/?modal=login` и `/?modal=register`
- Главная страница (`pages/index.tsx`) отслеживает query-параметр `modal` через `useRouter`
- При наличии параметра рендерится соответствующий компонент модалки поверх контента
- Закрытие окна (клик по затемнению или успешная авторизация) возвращает пользователя в исходное состояние через `shallow routing`

### Многоуровневая валидация

Валидация данных происходит на двух уровнях:

#### 1. Клиентская (мгновенная обратная связь)
- Пустые поля — проверка `trim()` перед отправкой
- Формат email — регулярное выражение `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- Совпадение паролей — проверка идентичности в форме регистрации
- Визуальная индикация — красная рамка на невалидном поле (`styles.inputError`)
- Сброс ошибки — при начале ввода в подсвеченном поле
- Отключение браузерной валидации — атрибут `noValidate` на форме

#### 2. Серверная (безопасность и бизнес-логика)
- Проверка существования пользователя в БД
- Валидация сложности пароля
- Защита от дублирования записей

### Безопасность обработки ошибок

Внешний учебный API (`wedev-api.sky.pro`) реализует **безопасную обработку ошибок авторизации**: для предотвращения перебора зарегистрированных email-адресов сервер возвращает **единый статус 400** без уточнения причины (почта не найдена / неверный пароль / пользователь уже существует).

На клиенте реализована соответствующая унифицированная обработка:
- При любой ошибке авторизации показывается сообщение: *"Неверные данные. Проверьте почту и пароль"*
- Подсвечивается поле email как наиболее вероятная причина
- Это соответствует современным стандартам безопасности (OWASP)

### Защита от повторной отправки

- Кнопка отправки блокируется через `disabled={isLoading}` во время запроса
- Предотвращает дублирование запросов к серверу
- Визуально показывает процесс загрузки ("Вход..." / "Регистрация...")

## 🚀 Локальный запуск
1. Установите зависимости:

```bash
    npm install
```
2. Запустите локальную MongoDB (порт 27017).
3. Создайте файл .env.local в корне проекта:
    - MONGODB_URI=mongodb://127.0.0.1:27017/skyfitness
    - MONGODB_DB=skyfitness
    - JWT_SECRET=your_secret_key_here

4. Запустите сервер разработки:
```bash
    npm run dev
```
5. Откройте http://localhost:3000
